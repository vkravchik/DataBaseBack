Потрібно розробити інформаційну систему для автоматизації автопідприємства міста. Система повинна передбачити режим ведення системного каталогу, що відображає маршрути громадського транспорту, та вантажного. Також ІС повинна містити інформацію про ремонт транспорту та бригади.

Кожен пасажирський маршрут, що зберігається у БД характеризується наступними параметрами:

-      id (int, primary key);

-      id_street (int, foreign key);

-      id_auto (int, foreign key);

-      passanger_count (int).

У місті ведеться також облік грузових перевезень, які характерезиються такими полями:

-      id (int, primary key);

-      point_start (varchar);

-      point_finish (varchar);

-      date_start (datetime);

-      date_finish (datetime);

-      id_auto (int, foreign key).

Для кожного виду транспорту, описуються унікальні характеристики характерні цьому типу. Для таблиці автобуси:

-        id (int, primary key)

-        id_auto (int, foreign key)

-        number_seated_passengers (int)

-        number_standing_passengers (int)

Для таблиці таксі:

-        id (int, primary key)

-        id_auto (int, foreign key)

-        number_passengers (int)

Для таблиці вантажний транспорт:

-        id (int, primary key)

-        id_auto (int, foreign key)

-        load_capacity (int)

Для таблиці спец техніка:

-        id (int, primary key)

-        id_auto (int, foreign key)

-        description (varchar)

 

Місто веде облік об’єктів гаражного підприємства до полів якого належать:

-                  id (int, primary key);

-                  name (varchar);

-                  street (varchar).

Підприємство займається ремонтом техніки:

-        id (int, primary key);

-        id_problem (int, foreign key);

-        id_auto (int, foreign key);

-        date (date);

-        id_personal (int, foreign key).

У разі повної поломки транспорту, підприємство має змогу списати його та купити новий:

-        id (int, primary key);

-        id_auto (int, foreign key);

-        date (date);

-        sale_buy (tiny_int);

-        price (int).

На фірмі працює персонал який об’єднується в групи і має бригадирів. Персонал має прив’язку до кваліфікації. Виходячи з цього отримаємо ще п’ять таблиці.

Категорії персоналу:

-        id (int, primary key);

-        name (varchar).

Персонал:

-        id (int, primary key);

-        name (varchar);

-        surname (varchar);

-        id_category_personal (int, foreign key).

Бригадир:

-        id (int, primary key);

-        name (varchar);

-        surname (varchar).

Бригада:

-        id (int, primary key);

-        name (varchar).

Водії:

-        id (int, primary key);

-        name (varchar);

-        surname (varchar);

-        id_auto (id, foreign key).

Групи:

-        id (int, primary key);

-        id_personal (int, foreign key);

-        id_drivers (int, foreign key);

-        id_brigada (int, foreign key);

-        id_brigadir (int, foreign key).

 

Потрібно передбачити наступні обмеження на інформацію в системі:

1)              Всі числові дані не повинні містити від’ємних чи символьних значень;

2)              За одним авто може бути закріплено кілька водіїв;

3)              Поля tinyint набувають значень 0 або 1, false & true відповідно;

4)              Облік числа перевезених пасажирів не може бути від’ємним;

5)              Вік водія не може бути меншим за 18 років.

З даною ІС мають працювати наступні групи користувачів:

1)              Адміністрація міста;

2)              Адміністрація фірми;

3)              Водії;

4)              Бригади.

При роботі з системою, адміністрація міста повинна мати доступ до:

1)              Інформації про ремонт техніки;

2)              Кількості перевезених пасажирів по маршрутах;

3)              Інформації про грузові перевезення;

4)              Об’єктів фірми.

У свою чергу водії повинні мати доступ лише до інформації, яка потрібна їм для здійснення тих чи інших перевезень.

Бригадири повинні мати доступ до:

1)              Інформації про об’єкти;

2)              Інформації по використанню спец техніки;

3)              Облік купленого та списаного транспорту;

4)              Обліку бригад

Адміністрація фірми має доступ до всіх даних ІС.
